# Airbnb Clone Backend - Use Case Diagram

---

## 🎯 Purpose

To document and visualize:

- Who uses the system (actors)
- What each actor can do (use cases)
- How actors interact with the system

This helps guide development, testing, and documentation efforts by clarifying expected behavior and responsibilities.

---

## 🧍‍♂️ Actors

1. **Guest** – A user who browses, searches, and books properties.
2. **Host** – A user who lists and manages rental properties.
3. **Admin** – A superuser who monitors and manages the entire platform.

---

## 📋 Key Use Cases

### For All Users

- Login
- Logout
- Update Profile
- View Notifications

### For Guests

- Search Properties
- Book Property
- Cancel Booking
- Leave Review
- Send Message to Host

### For Hosts

- Add Property
- Edit/Delete Property
- Manage Availability
- Respond to Reviews
- Receive Messages
- Accept/Decline Booking Requests
- Receive Payment

### For Admins

- Manage Users
- Manage Properties
- Manage Bookings
- Monitor Payments
- Resolve Disputes
